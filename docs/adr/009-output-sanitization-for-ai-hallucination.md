# ADR 009: Bot コメント出力のサニタイズ（AI ハルシネーション・フィッシング対策）

## Status
* Proposed

## Context (背景と課題)
`index.ts` および `utils/github.ts` では、LLMが生成した判定理由（`result.reasoning`）や修正案（`result.suggestion`）を、そのままの状態で GitHub PR のインラインコメントとして投稿 (`postOrUpdateComment`) しています。
LLMは「ハルシネーション（もっともらしい嘘）」を起こす性質があり、実在しないURLやライブラリ名を生成する可能性があります。攻撃者がプロンプト・インジェクション等を用いてAIを誘導し、悪意あるフィッシングURLや、攻撃者が仕掛けたマルウェアパッケージ名（Slopsquatting）を含む Markdown リンク・画像を Bot アカウントから投稿させるリスク（Markdown Injection）が存在します。開発者が Bot を信頼してこれをクリック・実行してしまうと、深刻なセキュリティインシデントに繋がります。

## Decision (決定事項)
GitHub Actions の Bot が投稿するコメント（特に LLM が生成した不定形のテキスト）に対して、最低限の **防衛的サニタイズ（Defensive Sanitization）** を実施します。

1.  **リンクと画像の無効化:** LLM の出力（`reasoning`）に含まれる Markdown 形式のリンク記法 `[text](url)` や画像記法 `![alt](url)`、および生URL（`http://` 等）を検出した場合、クリック可能な状態からプレーンテキスト（またはコードブロック内）へエスケープ変換する処理を挟みます。
2.  **Disclaimer の付与:** 投稿するコメントの末尾（シグネチャ付近）に、「このコメントはAIによって生成されたものであり、URLや提案されたコードを実行する前に人間による確認が必要です」という免責事項（Disclaimer）を必ず明記します。

## Consequences (結果・影響)
*   **[Good] 二次被害の防止:** 攻撃者による AI の踏み台化（フィッシングURLの拡散など）を水際で防ぐことができます。
*   **[Bad] UXの低下:** AIが本当に有用な参考リンク（例: 公式ドキュメントのURL）を生成してくれた場合でも、クリックできないテキストになってしまうため、開発者の利便性が少し下がります。
